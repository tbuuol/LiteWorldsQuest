Aktionen

Dies sind die bisher implementierten Aktionen. Jede Aktion kann weitere Unteraktionen haben:



  • Aktivierung – Der Kern von MoonBrix arbeitet mit 15 Bytes. Diese 15 Bytes, dargestellt als "00", eröffnen das Protokoll.
  • Abstecken – Domains und IPs können abgesteckt (geclaimt) werden.
  • Besitzer – Das Eigentumsrecht einer Domain kann übertragen werden.
  • Listentyp – Domains können als weiße oder schwarze Liste geführt werden.
  • Liste modifizieren – Adressen können zur Liste hinzugefügt oder entfernt werden.
  • IP-Namen – Jeder IP kann ein eindeutiger Name innerhalb der Domain zugewiesen werden.
  • Dateieinschreibung – Dateien (Text, Bilder, Audio, Video) können in die Blockchain geschrieben werden.
  • Ordner – Ordner können im Dateisystem erstellt oder gelöscht werden.
  • Dateien – Dateien können in Ordnern erstellt und einer Einschreibung zugeordnet werden.
  • Eigenschaft – Ähnlich einem Smart Contract; definiert Regeln für einen Tokenraum.
  • Token – Token können erzeugt, vernichtet, verschickt oder gelistet werden.

MoonBrix Info

Was ist MoonBrix?


MoonBrix nutzt den OP_RETURN-Code, um eine Infrastruktur direkt auf der Blockchain aufzubauen.

Durch das Anhängen von Bytes an Transaktionen und anschließendes Auslesen durch einen Interpreter entsteht eine Struktur, die verschiedene Dienste ermöglicht.

Diese Dienste umfassen Domains mit verschiedenen Optionen, IP-Bereiche, bei denen jede Domain ihren eigenen Bereich hat und IPs Namen erhalten können, sowie ein eigenes Dateisystem für jede IP.

Außerdem können Dateien ähnlich dem Ordinals-System ins Dateisystem eingebunden werden, und es gibt ein Token-System, das dem Omni Layer nachempfunden ist.



Auf welcher Blockchain läuft MoonBrix?


Bitcoin ist für dieses Protokoll zu limitiert und überlastet (siehe OmniLayer oder Ordinals).

Litecoin kann MoonBrix verarbeiten, abhängig von der Anzahl der Nutzer.

Aktuell wird MoonBrix auf Kotia entwickelt.

MoonBrix ist standardmäßig auf eine maximale Länge von 80 Bytes pro Transaktion ausgelegt, außer bei Dateieinschreibungen, die aktuell bis zu 10.000 Bytes umfassen.

Durch diese Konfiguration und die Möglichkeit, mehrere Blockchains in einer Instanz zusammenzufassen, ist MoonBrix sehr anpassungsfähig und bietet direkt nutzbare Funktionalität.

Technische Hinweise & Ausblick

Systemanforderungen


MoonBrix läuft aktuell ausschließlich unter Linux. Das ist keine Designentscheidung, sondern pure Notwendigkeit. Windows hat ein ziemlich... eigenwilliges Verhältnis zu seinen Sockets – sobald der Sync richtig Gas gibt, kippt das System reihenweise um. Unter Linux hingegen läuft der Prozess stabil, performant und mit deutlich besserem Speicherverhalten.

Langfristig ist ein Cross-Plattform-Support geplant, aber erst dann, wenn Windows aufhört, sich beim Socket-Handling wie ein beleidigtes Kind zu benehmen. Bis dahin gilt: Wer ernsthaft mit MoonBrix arbeitet, nutzt Linux – Punkt.

MoonBrix ist so konzipiert, dass mehrere Blockchains in einer Instanz parallel verwaltet werden können. Diese Architektur erlaubt es, Daten, Domains, IP-Bereiche, Token und Dateisysteme verschiedener Chains zu verbinden, ohne dass sie sich gegenseitig in die Quere kommen. Das Ziel: ein universelles, kettenübergreifendes Infrastruktur-Protokoll, das nicht nur Daten speichert, sondern auch Interaktionen ermöglicht.



Limitierungen


Das Herzstück bildet der Interpreter, der OP_RETURN-Transaktionen ausliest, analysiert und in verwertbare Strukturen übersetzt. Jede Aktion – ob Datei, Token oder Domain – ist ein kleines Puzzleteil in einem größeren System, das auf Einfachheit und Geschwindigkeit ausgelegt ist.

Der aktuelle Limitierungsrahmen beträgt 80 Bytes pro Standard-Transaktion, Dateieinschreibungen liegen bei bis zu 10.000 Bytes. Diese Grenzen wurden bewusst gesetzt, um den Netzwerk-Traffic niedrig zu halten und den Durchsatz stabil zu halten. Eine spätere Erhöhung über Chunking-Systeme ist denkbar.



Zukunftspläne


In Zukunft sind folgende Erweiterungen vorgesehen:

- Multi-Chain-Kommunikation (Litecoin, Kotia, evtl. Dogecoin)

- Erweiterte Tokenlogik mit Eigenschaften und Smart-Asset-Verhalten

- Dezentralisierte Ordnerstrukturen und dateibasierte Signaturen

- Vollautomatisierte Sync-Mechanismen zwischen Nodes

- Ein optionales Frontend für Entwickler mit Live-Datenansicht

MoonBrix versteht sich nicht als weiteres „Blockchain-Projekt“, sondern als Fundament. Eine Basis, auf der Dienste, Netzwerke und Applikationen entstehen können – robust, direkt und ohne Abhängigkeiten von externen Plattformen. Kein JSON-Overload, kein überkomplexes Framework. Nur klare Bytes, saubere Logik und ein Ziel: Kontrolle über die eigene Infrastruktur zurückholen.
